Prompt Engineering

3 days • Jul 13 (Mon) - Jul 15 (Wed) • No prior AI knowledge needed
What is this? The craft of writing precise instructions that get exactly the output you want from Claude / GPT / any LLM. Prompts are the "code" of the AI era.

Why should you learn this first? Every phase after this uses AI heavily (Claude Code, RAG, capstone). A great prompt turns a 4-hour task into 20 minutes. A bad prompt wastes tokens and time. This is the meta-skill that multiplies everything else.

Topics covered: 5-element prompt anatomy (Role, Context, Task, Format, Constraints); zero-shot; few-shot with examples; chain-of-thought reasoning; role prompting; structured JSON output; iterative refinement
Daily Schedule — Starting From Pure Basics
DAY 1Mon Jul 135-Element Anatomy + Zero-Shot
Concepts
  • The 5 elements of a strong prompt: Role, Context, Task, Format, Constraints
  • Zero-shot = ask directly, no examples — works when the task is common (translate, summarise, classify)
  • When zero-shot fails: ambiguous instructions, novel tasks, formatting-sensitive output
  • How to score a prompt: clarity + specificity + verifiability of output
  • Anti-patterns: vague roles ("be helpful"), missing constraints, no output format specified
Read / Watch

Anthropic Prompt Engineering docs (Intro + Basics sections); "ChatGPT Prompt Engineering" DeepLearning.AI Ch.1-2.

How to Practice

Write 5 zero-shot prompts for different tasks. Evaluate and score them 1–5 for clarity.

Mini Code Snippet
# Example prompt (5 elements) Role: You are a senior technical writer. Context: Users read release notes on mobile in <30s. Task: Rewrite the changelog below in 3 bullets. Format: Markdown bullets, no headings. Constr: Under 40 words total. No jargon. <changelog>{{input}}</changelog>
DAY 2Tue Jul 14Few-Shot + Chain-of-Thought
Concepts
  • Few-shot = 2–5 input/output examples shown before the real question
  • Examples must be diverse (edge cases) + short (token cost) + consistent format
  • Chain-of-Thought (CoT) = ask the model to reason step by step before answering
  • CoT shines on math, logic, multi-step debugging — not on lookups
  • Zero-shot CoT trick: just append "Let's think step by step" to a plain prompt
Read / Watch

Anthropic docs (Examples section); DeepLearning.AI Ch.3–4; read OpenAI "Chain of Thought" paper (abstract + examples only).

How to Practice

Take 3 complex tasks (code debug, reasoning, summarization) and write both few-shot and CoT prompts. Compare outputs.

Mini Code Snippet
# Few-shot classifier prompt Classify sentiment. Examples: "Loved the update" → POSITIVE "Battery drained fast" → NEGATIVE "Works ok I guess" → NEUTRAL Now classify: "{{text}}" # CoT variant: add "Let's think step by step" before answering
DAY 3Wed Jul 15Structured Outputs + Iterative Refinement + Role Prompting
Concepts
  • JSON output: describe the schema in the prompt + include "Return ONLY valid JSON"
  • For strict schemas use tool use / function calling instead of raw JSON
  • Role prompting: You are a senior SQL DBA. Reply ONLY with valid MySQL 8 queries
  • Iterative refinement loop: run → grade output → add missing constraint → rerun
  • Version your prompts (v1, v2, v3) — keep the diffs in a "prompt library" file
Read / Watch

Anthropic docs (Output Formatting, Role Prompting sections); review your Day 1–2 prompts.

How to Practice

Mini Code Snippet
